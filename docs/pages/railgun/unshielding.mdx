## From EOA

This transaction can be sent from an EOA. This can be the recipient itself, or a different EOA.
If the recipient is not an EOA, or does not have ETH yet, you can use a [relayer](#railgun-relayer) to submit the tx on your behalf.

:::note
Do note that this EOA will be the submitter of the tx, and thus linked to the recipient.
:::

```ts filename="vars.ts" twoslash
// @noErrors
const value = 10000000000000n; // 0.00001 ETH
const recipient = '0x1234567890123456789012345678901234567890' as `0x${string}`;
```

:::code-group

```ts twoslash [Ethers]
// @noErrors
import type { RailgunAccount } from '@kohaku-eth/railgun';
import type { Wallet } from 'ethers';

declare const account: RailgunAccount
declare const wallet: Wallet

const chainId = '11155111' as const;
const WETH = '0xfFf9976782d46CC05630D92EE39e704800271F7D' as const;
const value = 10000000000000n; // 0.00001 ETH
const recipient = '0x1234567890123456789012345678901234567890' as const;

const tx = await account.unshield(chainId, WETH, recipient, value); // [!code focus]

// Submit the tx using Ethers
await wallet.sendTransaction({
  to: tx.to,
  data: tx.data,
  value: tx.value,
  gasLimit: 6000000
});
```

// @noErrors
