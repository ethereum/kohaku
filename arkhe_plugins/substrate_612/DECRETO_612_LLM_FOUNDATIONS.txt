3. P1 — FOUNDATIONS (Fundamentos)
─────────────────────────────────────────────────────────────────────────────

  1.1 LLM Basics
      • Definição: Large Language Models — modelos de linguagem com
        bilhões/trilhões de parâmetros
      • Arquitetura base: Transformer (Vaswani et al., 2017)
      • Capacidades emergentes: in-context learning, chain-of-thought,
        few-shot prompting

  1.2 How AI Models Work
      • Forward pass: input → embedding → layers → output logits
      • Backward pass: gradient descent, backpropagation
      • Loss functions: cross-entropy, KL divergence
      • Optimization: Adam, AdamW, SGD with momentum

  1.3 Tokens
      • Unidade básica de processamento em LLMs
      • 1 token ≈ 0.75 palavras (inglês) ou 1-2 caracteres (chinês)
      • Exemplo: "ChatGPT is great" ≈ 6 tokens

  1.4 Tokenization
      • BPE (Byte-Pair Encoding): GPT-2, GPT-3, GPT-4
      • SentencePiece: LLaMA, PaLM, T5
      • Tiktoken: OpenAI's fast BPE tokenizer
      • Impacto: vocabulário, compressão, multilinguagem

  1.5 Context Windows
      • Definição: número máximo de tokens processados em uma passagem
      • Evolução: 2k (GPT-3) → 4k (GPT-4) → 128k (GPT-4 Turbo) → 1M+ (Gemini)
      • Técnicas de extensão: RoPE scaling, ALiBi, LongLoRA
      • Tradeoff: memória O(n²) com attention

  1.6 Embeddings
      • Representação vetorial densa de tokens/sentences
      • Dimensões: 768 (BERT-base) → 12288 (GPT-3) → 16384 (GPT-4)
      • Similaridade: cosine similarity, dot product
      • Aplicações: RAG, clustering, classification

  1.7 Transformers
      • Arquitetura: Encoder-Decoder (T5), Decoder-only (GPT), Encoder-only (BERT)
      • Componentes: Multi-Head Attention, FFN, Layer Norm, Residual Connections
      • Positional encoding: absolute (sinusoidal), relative (RoPE), rotary (LLaMA)

  1.8 Attention Mechanism
      • Self-attention: Q, K, V matrices
      • Scaled dot-product: Attention(Q,K,V) = softmax(QK^T/√d_k)V
      • Multi-head: h heads paralelos concatenados
      • Flash Attention: IO-aware algorithm (Dao et al., 2022)

  1.9 Parameters
      • Contagem: weights + biases em todas as camadas
      • Escalas: 7B (LLaMA-2) → 70B (LLaMA-2) → 405B (LLaMA-3) → 1T+ (rumored)
      • Densidade: dense (todos ativos) vs sparse (MoE)

  1.10 Training vs Inference
      • Training: forward + backward, atualização de weights, GPU cluster
      • Inference: forward only, KV cache, batching, streaming
      • Custo: training ~$100M+ (GPT-4), inference ~$0.01/1K tokens

  1.11 Open-Source vs Closed-Source
      • Open: LLaMA, Mistral, Qwen, DeepSeek — weights disponíveis
      • Closed: GPT-4, Claude, Gemini — API-only
      • Híbridos: Mixtral, Gemma — weights abertos com restrições

4. P2 — DATASETS & TRAINING (Dados e Treinamento)
─────────────────────────────────────────────────────────────────────────────

  2.1 SFT Datasets (Supervised Fine-Tuning)
      • Alpaca (52K instructions)
      • Dolly (15K human-generated)
      • OpenAssistant (161K messages)
      • ShareGPT (conversas reais com GPT-4)

  2.2 Instruction Tuning
      • Formato: {instruction, input, output}
      • Templates: Alpaca, Vicuna, ChatML
      • System prompts: role definition, constraints

  2.3 Preference Datasets
      • RLHF: {prompt, chosen, rejected}
      • Anthropic HH-RLHF (human feedback)
      • SHP (Stanford Human Preferences)
      • UltraFeedback (multi-model rankings)

  2.4 Synthetic Datasets
      • Self-Instruct: LLM gera instruções
      • Alpaca-GPT4: GPT-4 gera dados de treino
      • Evol-Instruct: evolução iterativa de complexidade
      • Constitutional AI: auto-correção via princípios

  2.5 Data Curation
      • Deduplication: MinHash, SimHash
      • Quality filtering: perplexity scoring, classifier-based
      • Domain balancing: mixture of sources
      • Toxicity removal: Perspective API, custom classifiers

  2.6 Dataset Cleaning
      • PII removal: regex, NER, manual review
      • Format normalization: JSONL, Parquet
      • Token length filtering: truncamento, splitting
      • Language detection: fastText, cld3

  2.7 Dataset Formatting
      • Conversational: ChatML, ShareGPT, Llama-2-chat
      • Completion: prompt-completion pairs
      • Preference: triples (prompt, chosen, rejected)
      • Packing: múltiplos exemplos em uma sequência

  2.8 Fine-Tuning Basics
      • Full fine-tuning: atualiza todos os parâmetros
      • Selective: atualiza apenas camadas específicas
      • Progressive: unfreeze gradualmente
      • Learning rates: 1e-5 a 1e-7 (menor que pretraining)

  2.9 Continued Pretraining
      • Domain adaptation: medicina, direito, código
      • Datasets especializados: PubMed, StackOverflow
      • Catastrophic forgetting: mitigation via replay
      • Learning rate: menor que fine-tuning

  2.10 Hallucination Reduction
      • RAG: retrieval-augmented generation
      • Tool use: calculators, search engines
      • Self-consistency: sampling múltiplo + votação
      • Fact-checking: NLI models, claim verification

5. P3 — FINE-TUNING (Ajuste Fino)
─────────────────────────────────────────────────────────────────────────────

  3.1 LoRA (Low-Rank Adaptation)
      • Hu et al., 2021
      • Decomposição de matrizes de update em low-rank
      • Rank r << d (dimensão original)
      • W' = W + BA (B: d×r, A: r×d)
      • Redução de parâmetros treináveis: ~99.9%

  3.2 QLoRA (Quantized LoRA)
      • Dettmers et al., 2023
      • 4-bit quantization (NormalFloat4)
      • Double quantization: nested quantization
      • Paged optimizers: offload para CPU
      • Treina 65B model em 48GB GPU

  3.3 DPO (Direct Preference Optimization)
      • Rafailov et al., 2023
      • Simplifica RLHF: preference loss direto
      • Não precisa de reward model separado
      • Loss: -log σ(β log π(y_w|x) / π_ref(y_w|x) - β log π(y_l|x) / π_ref(y_l|x))

  3.4 RLHF (Reinforcement Learning from Human Feedback)
      • InstructGPT, ChatGPT
      • 3 fases: SFT → Reward Model → PPO
      • PPO: Proximal Policy Optimization
      • KL penalty: evita drift do modelo base

  3.5 Quantization
      • Post-training: INT8, INT4, GPTQ, AWQ
      • Quantization-aware training (QAT)
      • GGUF: format para llama.cpp
      • Tradeoff: speed vs accuracy

  3.6 Model Checkpoints
      • Formats: .bin (PyTorch), .safetensors (HuggingFace)
      • Sharding: FSDP, DeepSpeed ZeRO
      • Resume training: optimizer state, scheduler
      • Versioning: DVC, MLflow

  3.7 Adapter Tuning
      • Adapter layers: bottleneck architecture
      • Parallel vs serial adapters
      • Compatibilidade: merge, unmerge
      • Multi-task: adapter composition

  3.8 GGUF Models
      • Format de Georgi Gerganov (llama.cpp)
      • Quantization: Q4_0, Q5_K_M, Q8_0
      • Metadata: tokenizer, special tokens
      • Inferência: CPU e GPU (Vulkan, CUDA, Metal)

6. P4 — INFERENCE & OPTIMIZATION (Inferência e Otimização)
─────────────────────────────────────────────────────────────────────────────

  4.1 KV Cache
      • Armazena Key e Value matrices para tokens anteriores
      • Reduz computação de O(n²) para O(n) por token
      • Memória: 2 × n_layers × n_heads × d_head × seq_len × batch_size × bytes
      • PagedAttention (vLLM): blocos contíguos de memória

  4.2 Flash Attention
      • Dao et al., 2022
      • IO-aware: minimiza HBM reads/writes
      • Tiling: blocos que cabem em SRAM
      • Speedup: 2-4× em GPUs A100, 7.6× em H100

  4.3 Speculative Decoding
      • Draft model pequeno + target model grande
      • Aceitação: verificação paralela de tokens
      • Speedup: 2-3× com draft 10× menor
      • Medusa, Lookahead Decoding: alternativas

  4.4 Inference Optimization
      • Continuous batching (vLLM, TGI)
      • Tensor parallelism: shards em múltiplas GPUs
      • Pipeline parallelism: layers em GPUs diferentes
      • Model parallelism: combinação de ambos

  4.5 Model Serving
      • Frameworks: vLLM, TGI, TensorRT-LLM, SGLang
      • Batching: static vs dynamic vs continuous
      • Scheduling: FCFS, priority, preemptive
      • Autoscaling: Kubernetes, Kserve, BentoML

  4.6 Batch Inference
      • Static batching: preenchimento (padding)
      • Dynamic batching: janela deslizante
      • Continuous batching: preemptivo, mais eficiente
      • Tradeoff: throughput vs latency

  4.7 GPU Basics
      • Arquitetura: CUDA cores, Tensor Cores, RT Cores
      • Memória: HBM2e, HBM3, GDDR6X
      • Interconnect: NVLink, NVSwitch, PCIe
      • Topologia: DGX, HGX, custom clusters

  4.8 VRAM Basics
      • Modelo 7B Q4: ~4GB
      • Modelo 70B Q4: ~40GB
      • KV cache: adiciona ~2GB por 1K tokens (7B)
      • Otimizações: gradient checkpointing, offloading

  4.9 Latency vs Quality Tradeoffs
      • Temperature: 0.0 (determinístico) → 1.0 (criativo)
      • Top-p (nucleus sampling): 0.1 → 1.0
      • Max tokens: curto (rápido) vs longo (completo)
      • Quantização: Q4 (rápido) vs FP16 (preciso)

7. P5 — LOCAL AI ECOSYSTEM (Ecossistema Local de IA)
─────────────────────────────────────────────────────────────────────────────

  5.1 llama.cpp
      • C/C++ inference engine por Georgi Gerganov
      • Quantização GGUF, CPU inference
      • Backends: CUDA, Metal, Vulkan, OpenCL
      • Bindings: Python, Node.js, Rust, Go

  5.2 Ollama
      • Wrapper user-friendly para llama.cpp
      • Model hub: pull, run, create Modelfile
      • REST API: /api/generate, /api/chat
      • Integração: LangChain, LlamaIndex

  5.3 vLLM
      • PagedAttention + continuous batching
      • Throughput máximo para serving
      • OpenAI-compatible API
      • Tensor parallelism, pipeline parallelism

  5.4 MLX
      • Framework Apple Silicon (M1/M2/M3)
      • Unified memory: RAM = VRAM
      • Quantização nativa, LoRA training
      • Swift bindings para apps iOS/macOS

  5.5 Hugging Face
      • Hub: 500K+ models, datasets, spaces
      • Transformers library: PyTorch, TensorFlow, JAX
      • Tokenizers: fast, Rust-based
      • Inference: pipeline, accelerate, optimum

  5.6 Unsloth
      • Fine-tuning 2x faster, 70% less memory
      • Flash Attention 2 integration
      • QLoRA, DPO, ORPO support
      • Free tier: Colab, Kaggle

  5.7 Axolotl
      • YAML-based fine-tuning configuration
      • Multi-GPU, FSDP, DeepSpeed
      • Datasets: JSONL, Parquet, HuggingFace
      • Logging: WandB, TensorBoard

  5.8 PEFT (Parameter-Efficient Fine-Tuning)
      • HuggingFace library
      • LoRA, IA³, Prefix Tuning, Prompt Tuning
      • Adapters: save, load, merge
      • Integration: Transformers, Accelerate

  5.9 TRL (Transformer Reinforcement Learning)
      • HuggingFace library
      • SFT, Reward Modeling, PPO, DPO
      • Online DPO, RLOO
      • Integration: PEFT, Accelerate

8. P6 — RAG & MEMORY (RAG e Memória)
─────────────────────────────────────────────────────────────────────────────

  6.1 RAG (Retrieval-Augmented Generation)
      • Lewis et al., 2020
      • Arquitetura: retrieve → augment → generate
      • Dense retrieval: DPR, Contriever, E5
      • Sparse retrieval: BM25, TF-IDF
      • Hybrid: combinação de ambos

  6.2 Vector Databases
      • FAISS (Facebook): CPU/GPU, IVF, HNSW
      • Pinecone: managed, metadata filtering
      • Weaviate: GraphQL, modular AI
      • Chroma: local-first, embeddings
      • Qdrant: Rust-based, filtering
      • Milvus: distributed, cloud-native

  6.3 Chunking
      • Fixed-size: 512 tokens, overlap 50
      • Semantic: sentence/paragraph boundaries
      • Recursive: hierarchical splitting
      • Agentic: LLM decides chunk boundaries

  6.4 Retrieval Pipelines
      • Pre-filtering: metadata, date, source
      • Re-ranking: cross-encoders (ColBERT, monoT5)
      • Fusion: RRF (Reciprocal Rank Fusion)
      • Query expansion: HyDE, pseudo-relevance

  6.5 AI Memory Systems
      • Short-term: context window, KV cache
      • Long-term: vector DB, knowledge graph
      • Episodic: session history, user preferences
      • Semantic: concept extraction, ontology

  6.6 Semantic Search
      • Dense: embeddings + cosine similarity
      • Sparse: BM25 + keyword matching
      • Hybrid: α·dense + (1-α)·sparse
      • Cross-encoder: re-ranking preciso

10. P8 — MODEL TYPES (Tipos de Modelos)
─────────────────────────────────────────────────────────────────────────────

  8.1 VLMs (Vision-Language Models)
      • CLIP: contrastive learning, image-text
      • LLaVA: LLM + vision encoder
      • GPT-4V: multimodal understanding
      • Applications: OCR, chart QA, video analysis

  8.2 SLMs (Small Language Models)
      • <7B parameters
      • Phi-3, Gemma-2B, Qwen2-1.5B
      • Edge deployment: mobile, IoT
      • Distillation: knowledge transfer de LLMs

  8.3 Dense Models
      • Todos os parâmetros ativos em cada forward pass
      • GPT-4, LLaMA-2, Claude
      • Predictable: latency constante
      • Memory: linear com tamanho

  8.4 MoE (Mixture of Experts)
      • Shazeer et al., 2017
      • Sparse activation: ~10-20% dos parâmetros ativos
      • Routing: top-k experts por token
      • Exemplos: Mixtral 8x7B, GPT-4 (rumored), DeepSeek-V2
      • Tradeoff: throughput maior, memory maior

  8.5 Coding Models
      • Especializados em geração de código
      • CodeLlama, StarCoder, DeepSeek-Coder
      • Fill-in-the-Middle (FIM): prefix + suffix → middle
      • Benchmarks: HumanEval, MBPP, SWE-bench

  8.6 Reasoning Models
      • Otimizados para raciocínio lógico/matemático
      • o1, o3 (OpenAI): chain-of-thought interno
      • DeepSeek-R1: RL para reasoning
      • Test-time compute: mais tokens = melhor resposta

11. P9 — DEPLOYMENT (Deploy)
─────────────────────────────────────────────────────────────────────────────

  9.1 Local Inference
      • Hardware: GPU (NVIDIA), NPU (Apple), TPU (Google)
      • Frameworks: llama.cpp, vLLM, Ollama
      • Quantização: Q4, Q5, Q8 para VRAM limitado
      • Streaming: token-by-token output

  9.2 On-Device AI
      • Mobile: Core ML (iOS), NNAPI (Android)
      • Edge: Coral, Jetson, Raspberry Pi
      • Constraints: <2GB RAM, <1W power
      • Modelos: MobileLLM, Phi-3-mini, Gemma-2B

  9.3 API Serving
      • OpenAI-compatible: /v1/chat/completions
      • Streaming: Server-Sent Events (SSE)
      • Rate limiting: tokens/min, requests/min
      • Authentication: API keys, OAuth

  9.4 Cloud GPUs
      • Providers: AWS (p4d, p5), GCP (A100, H100), Azure
      • Serverless: Lambda, Cloud Functions (limitado)
      • Reserved: spot instances, savings plans
      • Cost: $2-8/hour (A100), $8-30/hour (H100)

  9.5 Edge AI Basics
      • Latency: <100ms para real-time
      • Bandwidth: minimizar data transfer
      • Privacy: dados não saem do dispositivo
      • Updates: OTA model updates

      • Automated: benchmarks, perplexity
      • Human: ratings, error analysis
      • A/B testing: statistical significance
